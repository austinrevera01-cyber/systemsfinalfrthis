## Contribution Model & Guidelines

1.  **Branching and Pull Requests**
    -   Use feature branches: `feature/...` or `fix/...`
    -   Create PRs before merging to `main`
    -   Use descriptive titles and link related issues
2.  **Coding Standards**
    -   MATLAB code must be commented and consistent
    -   P‑code files must include version, author, and date notes
    -   Documentation must maintain revision history
3.  **Issue Tracking**
    -   Create issues for bugs, tasks, enhancements
    -   Assign owners and due dates
    -   Use labels: `bug`, `enhancement`, `documentation`, etc.
4.  **Documentation**
    -   Datasheets describe system components
    -   `Fall_25_3140_Final_Project.pdf` is primary report
    -   Agents maintain their own sections
5.  **Review and Approval**
    -   Major changes require at least one reviewer approval
